## 1. Анализ аудитории и конверсии

### 1.1 Источники трафика на лендинг

| Источник | Доля (ориентир) | Характеристика |
|----------|-----------------|----------------|
| ASO (App Store / Google Play) → переход на сайт | 15% | Пользователь изучил приложение в маркете и хочет узнать больше до установки |
| Органический поиск (Google/Яндекс) | 35% | Запросы "квиз факты", "факты мифы приложение", "quiz myths app" |
| Социальные сети (TikTok, Instagram Reels, VK Clips) | 20% | Ролики с геймплеем → ссылка в описании |
| Прямые ссылки / QR-коды | 10% | Офлайн-реклама, рекомендации друзей |
| Реферальные ссылки (InfluencerMarketing) | 15% | Telegram-каналы про науку, факты, психологию |
| Платная реклама (Meta Ads, Яндекс.Директ) | 5% | Ретаргетинг и cold audiences |

**Ключевой вывод:** основная масса пользователей приходит уже тёплой — они либо видели геймплей, либо ищут нечто конкретное. Лендинг должен за 5 секунд ответить: "Это интересно, понятно как работает, скачать легко".

### 1.2 Портреты целевой аудитории

#### Портрет A — "Любитель знаний" (RU, 22–35 лет)
- Подписан на паблики "Интересные факты", "Наука и жизнь", "Загадки истории"
- Регулярно играет в мобильные игры (Wordle, "Эрудит")
- Боль: "Хочу развиваться, но нет времени читать длинные статьи"
- Желание: короткий контент, который реально чему-то учит
- Мотивация скачать: новые факты каждый день + объяснения к ответам

#### Портрет B — "Конкурентный геймер" (RU/EN, 18–28 лет)
- Играет в Trivia Crack, QuizUp, "Кто умнее школьника"
- Ценит рейтинги и стрики
- Боль: "Надоели одни и те же вопросы, хочу что-то реальное, а не выдуманное"
- Желание: честный рейтинг, ежедневный вызов, возможность победить
- Мотивация скачать: leaderboard + ежедневный streak

#### Портрет C — "Просветитель" (EN, 25–40 лет)
- Любит делиться интересным в чатах
- Ценит источники и объяснения
- Боль: "Слышу много мифов, хочу уметь их разоблачать"
- Желание: факты с объяснениями из проверенных источников
- Мотивация скачать: credibility контента + fun factor

### 1.3 Психологические триггеры конверсии

| Триггер | Реализация на лендинге |
|---------|------------------------|
| **Social proof** | Статистика: X фактов, X категорий, X пользователей |
| **Curiosity gap** | "Знаешь ли ты, что..." — пример утверждения прямо на Hero |
| **Streak / habit loop** | Подчеркнуть ежедневный набор, "не пропусти сегодняшний" |
| **FOMO** | "Новый набор каждый день — возвращайся!" |
| **Simplicity** | "3 секунды — один свайп" — максимально низкий порог входа |
| **Authority** | "Объяснения из проверенных источников" |

### 1.4 Best practices мобильных приложений (Duolingo, Trivia Crack, QuizUp)

**Что работает:**

1. **Анимированный мокап** — показывает механику без объяснений. Пользователь видит свайп → понимает игру за 2 секунды.
2. **CTA выше fold** — кнопки "Скачать" видны без скролла на мобильном.
3. **Конкретные цифры в social proof** — "500 000 фактов" работает лучше, чем "много фактов".
4. **Видимая механика** — секция "Как это работает" с иконками вместо текста.
5. **Двойной CTA** — в Hero и в конце страницы (для прокрутивших).
6. **Рейтинг магазинов** — звёзды App Store/Google Play как якорь доверия.
7. **Mobile-first дизайн** — 70%+ трафика с телефонов, CTA-кнопки должны быть thumb-friendly (min 48px).

**Что не работает:**
- Длинное описание в Hero
- Скриншоты вместо живого мокапа
- Один CTA на всю страницу
- Медленная загрузка (> 3 сек — потеря 53% пользователей, Google данные)

### 2.8 Canonical и hreflang стратегия

- Канонический URL главной страницы: `https://factfront.org/` (с трейлинг-слешем)
- Язык переключается через `?lang=en` — это query param, не отдельный путь
- `<link rel="canonical">` всегда указывает на `https://factfront.org/` (без `?lang=`)
- Для SEO: JS рендерит содержимое на нужном языке, но мета-теги переключаются через JS при изменении языка (`document.querySelector('meta[name=description]').setAttribute('content', ...)`)
- Stage полностью закрыт через `Disallow: /` в robots.txt

## 3. Полная структура лендинга

### 3.0 Технические требования к файлу

- Один файл `landing/index.html`
- Все стили инлайн в `<style>`
- JS инлайн в `<script>` в конце `<body>`
- Шрифт Nunito через Google Fonts CDN: `https://fonts.googleapis.com/css2?family=Nunito:wght@400;600;700;800;900&display=swap`
- Без внешних JS-библиотек (чистый DOM API)
- Размер без изображений: цель < 150KB (gzip), максимум 200KB
- Поддержка: Chrome 90+, Safari 14+, Firefox 88+, Samsung Internet 14+

### 3.2 Логика авто-определения языка и темы (JS)

```javascript
// Определение языка
const savedLang = localStorage.getItem('ff_lang');
const browserLang = navigator.language || navigator.userLanguage;
const urlLang = new URLSearchParams(window.location.search).get('lang');
const lang = urlLang || savedLang || (browserLang.startsWith('ru') ? 'ru' : 'en');

// Определение темы
const savedTheme = localStorage.getItem('ff_theme');
const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
const theme = savedTheme || (prefersDark ? 'dark' : 'light');

// Применение
document.documentElement.lang = lang;
document.documentElement.setAttribute('data-theme', theme);
```

### 4.5 Изменения в `web/nginx.conf`

**Текущий конфиг** обрабатывает SPA полностью через `try_files $uri $uri/ /index.html`. После изменения корень `/` должен отдавать лендинг, а `/admin` — SPA.

```nginx
# НОВЫЙ web/nginx.conf

server {
    listen 80;
    server_name _;

    # Gzip compression
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript image/svg+xml;
    gzip_min_length 256;

    # === ЛЕНДИНГ ===
    # Корневой маршрут → статический лендинг
    location = / {
        root /usr/share/nginx/landing;
        try_files /index.html =404;
        add_header Cache-Control "no-cache, no-store, must-revalidate";
    }

    # === РЕДИРЕКТЫ (обратная совместимость) ===
    # Редирект старого /login на /admin/login
    location = /login {
        return 301 /admin/login;
    }

    # === ADMIN SPA ===
    location /admin {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /admin/index.html;
    }

    # Статические ассеты SPA (хешированные имена, долгий кэш)
    location /assets/ {
        root /usr/share/nginx/html;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Прямой доступ к index.html SPA — no cache
    location = /admin/index.html {
        root /usr/share/nginx/html;
        add_header Cache-Control "no-cache, no-store, must-revalidate";
        add_header Pragma "no-cache";
        add_header Expires 0;
    }

    # === API PROXY ===
    location /admin/ {
        # Этот блок нужен только для API вызовов /admin/api/...
        # Роутинг к серверу через nginx-proxy.conf (внешний nginx)
        # Внутренний nginx только отдаёт SPA
        root /usr/share/nginx/html;
        try_files $uri $uri/ /admin/index.html;
    }

    location /api/ {
        proxy_pass http://server:3001/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # === PUBLIC ROUTES ===
    # /support — страница поддержки для пользователей (SPA)
    location /support {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /admin/index.html;
    }
}
```

**Примечание об архитектуре:** `web` контейнер будет содержать:
- `/usr/share/nginx/html/` — собранный React SPA
- `/usr/share/nginx/landing/index.html` — лендинг
