# Фаза 1: Креативный директор — Результаты

## 1. Генерация идей (10 штук)

---

### Идея 1: QuizDash — Ежедневная викторина

**Описание:** Мобильная викторина с ежедневными тематическими наборами вопросов (10 вопросов в день). Пользователи отвечают на вопросы из разных категорий (наука, история, география, поп-культура, логика), зарабатывают очки, поддерживают стрики и соревнуются в глобальном рейтинге.

**Почему откроют каждый день:** Новые вопросы каждый день, стрик-система (потеря стрика мотивирует), ежедневный рейтинг, желание проверить свои знания, уведомление «Твоя викторина ждёт!».

**Где реклама:** Interstitial между раундами (каждые 5 вопросов), баннер на экране результатов, rewarded video для подсказки или дополнительной попытки.

**Backend:** Да — хранение вопросов, глобальные рейтинги, ежедневные наборы, статистика. NestJS раздаёт контент и управляет рейтингами.

---

### Идея 2: HabitFlow — Трекер привычек со стриками

**Описание:** Минималистичный трекер привычек с красивой визуализацией прогресса (тепловая карта как в GitHub). Пользователи создают привычки, отмечают выполнение, смотрят статистику и получают мотивирующие уведомления.

**Почему откроют каждый день:** Необходимость отмечать привычки ежедневно, страх потерять стрик, визуальный прогресс, утренние/вечерние напоминания.

**Где реклама:** Баннер на главном экране внизу, interstitial при переходе к статистике (не чаще 1 раза в 3 мин), rewarded video для разблокировки расширенной аналитики.

**Backend:** Нет — все данные локально (AsyncStorage). Возможно в будущем: синхронизация между устройствами.

---

### Идея 3: BreathEasy — Дыхательные упражнения

**Описание:** Приложение с набором дыхательных техник (4-7-8, бокс-дыхание, дыхание Вима Хофа и др.) с красивыми анимациями, звуками природы и персональными программами. Помогает при стрессе, бессоннице, тревоге.

**Почему откроют каждый день:** Ежедневная рутина для снятия стресса, напоминания о дыхательных паузах, стрики дней практики, программы на 7/14/30 дней.

**Где реклама:** Баннер на главном экране, interstitial между сессиями, rewarded video для разблокировки премиум-техник или звуков.

**Backend:** Нет — все техники и программы зашиты в приложение. Обновление контента через OTA.

---

### Идея 4: SpinDecide — Колесо решений

**Описание:** Кастомизируемое колесо фортуны для принятия решений. Пользователи создают колёса (что поесть, куда пойти, что посмотреть, игры для компании), крутят и делятся результатами. Готовые шаблоны колёс для популярных сценариев.

**Почему откроют каждый день:** Решение бытовых вопросов (что приготовить, что делать), использование в компании друзей, создание новых колёс, шаринг в соцсети.

**Где реклама:** Interstitial после каждого 3-го спина, баннер внизу экрана, rewarded video для анимации «золотого спина» (визуальный эффект).

**Backend:** Нет — все колёса хранятся локально. Опционально: галерея популярных колёс (тогда backend).

---

### Идея 5: WordMaster — Ежедневная словесная головоломка

**Описание:** Набор словесных мини-игр: анаграммы, составление слов из букв, поиск слов, цепочки слов. Каждый день новый набор. Поддержка русского и английского — одновременно учишь слова.

**Почему откроют каждый день:** Новые головоломки каждый день, стрик, соревнование с друзьями, тренировка словарного запаса, привычка решать «ежедневку» (как Wordle).

**Где реклама:** Interstitial между мини-играми, баннер на экране результатов, rewarded video для подсказки.

**Backend:** Минимальный — сервер генерирует/раздаёт ежедневные наборы и ведёт лидерборд. Можно обойтись без backend, если наборы зашиты + OTA обновления.

---

### Идея 6: FocusBee — Pomodoro-таймер с геймификацией

**Описание:** Таймер продуктивности по методу Pomodoro с геймификацией: за каждый завершённый фокус-сеанс пользователь «выращивает» виртуального персонажа (пчела строит соты). Статистика за день/неделю/месяц.

**Почему откроют каждый день:** Использование для работы/учёбы каждый день, прогресс персонажа, статистика продуктивности, уведомления «Пора сфокусироваться!».

**Где реклама:** Баннер на экране статистики, interstitial при завершении 25-мин сеанса (в перерыве), rewarded video для кастомизации персонажа.

**Backend:** Нет — всё локально. Данные в AsyncStorage.

---

### Идея 7: MoodJournal — Дневник настроения

**Описание:** Ежедневный трекер настроения с мини-журналом. Пользователь выбирает настроение, описывает день в 1-3 предложения, приложение строит графики эмоционального состояния и выявляет паттерны.

**Почему откроют каждый день:** Ежевечерняя рефлексия, наблюдение за паттернами настроения, напоминания «Как прошёл твой день?», стрики.

**Где реклама:** Баннер на главном экране, interstitial при открытии еженедельного отчёта, rewarded video для экспорта данных.

**Backend:** Нет — все данные строго локально (приватная информация). Без сервера.

---

### Идея 8: BrainPulse — Тренировка мозга

**Описание:** 5-минутные ежедневные тренировки мозга: память, внимание, скорость реакции, логика, пространственное мышление. 5 мини-игр в день, прогресс и статистика когнитивных навыков.

**Почему откроют каждый день:** Ежедневная тренировка (5 мин — не лениво), прогресс когнитивных навыков на графике, сравнение с «нормой» для возраста, уведомления.

**Где реклама:** Interstitial между мини-играми (после 2-й и 4-й), баннер на экране результатов, rewarded video для разблокировки бонусной игры.

**Backend:** Минимальный — лидерборды и статистика по возрастным группам. Можно обойтись без backend на старте.

---

### Идея 9: WaterDrop — Трекер воды

**Описание:** Красивый трекер потребления воды с анимациями наполнения стакана/бутылки. Умные напоминания с учётом погоды и активности. Виджеты для быстрой отметки.

**Почему откроют каждый день:** 6-8 раз в день отмечают выпитую воду, напоминания «Пора попить!», визуальный прогресс к дневной цели, стрики.

**Где реклама:** Баннер на главном экране, interstitial при достижении дневной цели (праздничный), rewarded video для расширенной статистики.

**Backend:** Нет — все данные локально.

---

### Идея 10: StepQuest — Шагомер с квестами

**Описание:** Шагомер, превращающий ходьбу в приключение. Шаги конвертируются в «километры путешествия» по виртуальным маршрутам (Москва → Петербург, Нью-Йорк → Лос-Анджелес). На пути открываются факты о местах.

**Почему откроют каждый день:** Проверка шагов, прогресс по маршруту, открытие новых фактов о местах, ежедневные челленджи, уведомления «Осталось 2000 шагов до Твери!».

**Где реклама:** Баннер на главном экране, interstitial при достижении чекпоинта маршрута, rewarded video для разблокировки премиум-маршрутов.

**Backend:** Минимальный — маршруты и факты. Можно зашить в приложение + OTA.

### ТОП-1: WordMaster — Ежедневная словесная головоломка

#### Конкуренты

| Приложение | Рейтинг | Скачивания | Платформы |
|-----------|---------|------------|-----------|

| CodyCross | 4.6 | 50M+ | iOS, Android |
| WordBrain | 4.5 | 50M+ | iOS, Android |
| Тетрис слов | 4.4 | 5M+ | iOS, Android |

#### Негативные отзывы конкурентов (что бесит)
- Слишком много рекламы (каждые 2 минуты interstitial)
- Энергия / жизни — искусственное ограничение игрового времени
- Одинаковый тип головоломок — быстро надоедает
- Нет русского языка (для Wordle и западных клонов)
- Плохая работа оффлайн
- Навязчивая подписка / IAP

#### Сильные стороны конкурентов
- Огромные базы слов и уровней
- Красивый полированный UI
- Социальные фичи (поделиться результатом)
- Ежедневные задания (Wordle-формат — один раз в день, все решают одно и то же)

#### UVP (Unique Value Proposition)
**Несколько типов словесных мини-игр в одном приложении** (анаграммы + составление слов + цепочки + поиск) вместо одного формата. **Двуязычный** (RU + EN) с первого дня. **Без энергий и жизней** — играй сколько хочешь, реклама ненавязчивая. Ежедневный набор + бесконечный режим.

#### SWOT-анализ
| | |
|---|---|
| **Strengths** | Мультиформат (разнообразие не даёт скучать), двуязычность, без IAP-паулов, ежедневный + бесконечный режим, свежий контент через OTA |
| **Weaknesses** | Один разработчик — ограничение скорости добавления контента. Нет бренда. Маленький маркетинговый бюджет |
| **Opportunities** | Рынок словесных игр стабильно растёт. Мало качественных двуязычных приложений. Wordle создал привычку «ежедневной словесной игры». RuStore — слабая конкуренция |
| **Threats** | NY Times Wordle доминирует. Крупные студии могут скопировать формат. Необходимость постоянного обновления базы слов |

| | |
|---|---|
| **Strengths** | Очень простая разработка, не нужен backend, яркие анимации, виральный (шаринг в stories), библиотека шаблонов |
| **Weaknesses** | Низкий retention (открыл → покрутил → закрыл), мало поводов возвращаться ежедневно, ограниченная глубина приложения |
| **Opportunities** | Конкуренты имеют устаревший дизайн — можно выделиться качеством. Virale potential через TikTok/Reels |
| **Threats** | Слишком простое приложение — легко скопировать. Низкий DAU = мало рекламного дохода |

|------|-----------|-------------|--------|---------------------|
| Отклонение в App Store за «спам» (похожесть на Wordle) | Рыночный | Низкая | Высокий | Достаточная дифференциация: мульти-формат (не только Wordle), уникальный UI, оригинальное название |
| Отклонение в Google Play | Рыночный | Низкая | Высокий | Соответствие всем политикам, корректный Ads declaration, уникальные ассеты |
| Отклонение в RuStore | Рыночный | Низкая | Средний | Русскоязычное описание, соответствие требованиям, Yandex Ads SDK |

| Сервер недоступен — приложение не работает | Технический | Средняя | Высокий | Кэширование наборов на клиенте (последние 3-5 дней). Оффлайн-режим с кэшированным контентом. Graceful degradation |

| Выгорание соло-разработчика | Операционный | Средняя | Высокий | Чёткие спринты (1 неделя). MVP за 6-8 недель. Автоматизация (EAS Build, CI/CD). Не перфекционизм — ship fast, iterate |
| Нехватка качественного познавательного контента | Операционный | Средняя | Высокий | Использовать открытые словари (OpenCorpora для RU, NLTK для EN) как базу. Факты курировать вручную через админку. Заготовить 60 дней контента до запуска. Массовый импорт через CSV |
| GDPR / 152-ФЗ | Юридический | Низкая | Средний | Минимум персональных данных — не требуем регистрацию. Все данные на устройстве. Privacy Policy на сайте. Если рейтинг — только nickname |
| COPPA (дети до 13) | Юридический | Низкая | Высокий | Возрастной рейтинг 12+ или «для всех». Не таргетировать детей. Не собирать данные. Декларировать в сторах |
| Крупный конкурент копирует мультиформат | Рыночный | Средняя | Средний | Быстро наращивать аудиторию. Фокус на нишу двуязычных пользователей (RU+EN). Постоянное обновление контента |
| Падение интереса к словесным играм | Рыночный | Низкая | Средний | Словесные игры — «вечнозелёная» категория (кроссворды существуют 100+ лет). Wordle только усилил интерес |
| Технические проблемы с рекламными SDK | Технический | Средняя | Средний | Абстракция AdManager с fallback. Тестирование на реальных устройствах. Graceful degradation (при ошибке рекламы — просто не показывать) |
| AI генерирует недостоверные факты (галлюцинации) | Операционный | Высокая | Высокий | AI обязан указывать источник (Wikipedia, книги, энциклопедии). | Пул вопросов исчерпан для активного пользователя | Операционный | Средняя | Высокий | Минимум 500+ вопросов к запуску. AI-генерация позволяет быстро наращивать пул. Мониторинг остатка уникальных вопросов на дашборде |

#### Расчёт: сколько DAU нужно для $300/мес

**Формула:** Доход = DAU × Sessions/Day × Ads/Session × eCPM / 1000

**Допущения:**
- Средняя сессия: 1.5 раза в день
- Рекламных показов за сессию: 3 (1 баннер + 1 interstitial + 1 rewarded)
- Итого: 4.5 ad impressions / DAU / день

**Сценарий 1 (Международная аудитория, blend eCPM = $3.00):**
- $10/день ÷ ($3.00 × 4.5 / 1000) = $10 ÷ $0.0135 = **~740 DAU**

**Сценарий 2 (Смешанная RU + международная, blend eCPM = $2.00):**
- $10/день ÷ ($2.00 × 4.5 / 1000) = $10 ÷ $0.009 = **~1,100 DAU**

**Сценарий 3 (Преимущественно СНГ, blend eCPM = $1.00):**
- $10/день ÷ ($1.00 × 4.5 / 1000) = $10 ÷ $0.0045 = **~2,200 DAU**

**Вывод:** Реалистичная цель — **1,000-2,000 DAU** для достижения $300/месяц. Это достижимо через ASO + органический рост за 3-6 месяцев для качественного приложения в нише словесных игр.

#### Баланс между доходом и UX
- **НЕ показывать** interstitial во время игры (только между играми)
- **НЕ показывать** рекламу при первом запуске (первые 3 игры без рекламы)
- **Rewarded ТОЛЬКО** по желанию пользователя — никогда принудительно
- **Banner** только внизу, не поверх игрового поля
- **Частотный лимит** interstitial: макс 1 раз в 2 минуты, макс 10 в день
- **Нативный стиль** рекламного контейнера — мягкие цвета, скруглённые углы

### Нужен ли Backend?

**Решение: ДА, с первого дня (MVP)**

Backend — ключевая часть продукта, потому что контент (вопросы, задания, познавательные факты) должен управляться централизованно и обновляться без пересборки приложения.

**MVP (v1.0) — NestJS + PostgreSQL + Prisma:**
- **Управление контентом**: CRUD для вопросов, заданий, наборов, категорий, познавательных фактов
- **AI-генерация контента**: интеграция с LLM (Claude API / OpenAI API) для генерации вопросов, фактов и заданий прямо из админки. Владелец нажимает «Сгенерировать» → AI предлагает пачку вопросов → владелец валидирует, редактирует, одобряет или отклоняет каждый. **ВАЖНО**: AI обязан указывать источник для каждого факта (Wikipedia, научные книги, энциклопедии). Факты ни в коем случае НЕ должны быть выдуманы — только проверяемые данные из достоверных источников
- **AI-генерация иллюстраций**: интеграция с моделью генерации изображений (DALL-E / Stable Diffusion API) для автоматического создания тематических иллюстраций к вопросам. Владелец может перегенерировать или загрузить свою картинку
- **Ежедневные наборы**: формирование и публикация ежедневных тематических наборов через админку
- **API для мобильного приложения**: выдача наборов, вопросов, проверка ответов
- **Прогресс пользователя**: хранение стриков, статистики, пройденных наборов (по device ID или аккаунту)
- **Система антиповторов**: сервер отслеживает, какие вопросы/факты пользователь уже видел. Если ответил правильно — вопрос больше НИКОГДА не показывается. Если ответил неправильно — вопрос может повториться, но не ранее чем через N дней (spaced repetition light)
- **Глобальные лидерборды**: ежедневные / еженедельные рейтинги
- **Кэширование на клиенте**: загруженные наборы работают оффлайн, синхронизация при подключении
- **Push-уведомления**: отправка через FCM («Новый набор готов!», «Твой стрик под угрозой!»)
- **Хранение изображений**: S3-совместимое хранилище (Cloudflare R2 / AWS S3 / Supabase Storage) для иллюстраций к вопросам

**Почему не локально:**
- Контент должен быть интересным и познавательным — это требует курирования через админку
- AI-генерация позволяет быстро создавать большие объёмы контента, но каждый вопрос проходит ручную модерацию
- Система антиповторов требует серверного хранения истории пользователя
- Ежедневные наборы должны быть гибкими (тематические дни, праздники, события)
- Аналитика: какие вопросы слишком лёгкие/сложные, где пользователи бросают

**Расходы на сервер:** Railway / Render free tier на старте → $5-10/мес при росте. PostgreSQL — Supabase / Neon free tier. AI API — ~$5-15/мес на генерацию контента (не в реальном времени, только при создании). Хранилище изображений — Cloudflare R2 (10 GB бесплатно).

### Нужен ли Web?

**Решение: ДА — Админ-панель для управления контентом (MVP)**

**Web-админка (React + Vite) — apps/web/ — ТОЛЬКО для владельца:**

Админ-панель — это приватное веб-приложение, доступ к которому имеет только владелец (один аккаунт, JWT-авторизация, возможно IP-whitelist).

- **AI-генерация вопросов**:
  - Кнопка «Сгенерировать пачку» → задаёшь параметры (категория, сложность, язык, количество) → AI генерирует 10-50 вопросов с фактами
  - AI обязан для каждого факта указать **источник** (ссылка на Wikipedia, название книги/энциклопедии, научная статья)
  - Каждый сгенерированный вопрос отображается в статусе «На модерации» с указанием источника
  - Владелец просматривает факт, проверяет источник (кликабельная ссылка), редактирует, одобряет (✓) или отклоняет (✗)
  - Только одобренные и верифицированные вопросы попадают в пул для пользователей
  - **Никаких выдуманных фактов** — только достоверная информация из проверяемых источников
- **AI-генерация иллюстраций**:
  - Для каждого вопроса — кнопка «Сгенерировать картинку» → AI создаёт тематическую иллюстрацию
  - Предпросмотр, возможность перегенерировать или загрузить свою картинку
  - Стиль иллюстраций единый (задаётся через system prompt / style preset)
- **Управление вопросами и заданиями**: создание вручную, редактирование, удаление, массовый импорт CSV, фильтры по статусу/категории/сложности
- **Управление ежедневными наборами**: формирование наборов из пула одобренных вопросов, назначение даты публикации, привязка темы дня, предпросмотр набора
- **Управление познавательным контентом**: факты, этимология, пояснения к каждому вопросу/слову
- **Управление категориями и тегами**: наука, история, география, языки, логика и т.д.
- **Статистика**: популярность вопросов, процент правильных ответов, сложность, DAU/retention дашборд, количество контента в пуле (одобрено/на модерации/отклонено)
- **Авторизация**: JWT, единственный аккаунт владельца, опционально IP-whitelist или 2FA

**Публичная веб-версия игры: НЕТ для MVP, ВОЗМОЖНО для v2.0**
- В будущем можно добавить веб-версию как воронку привлечения → «Скачай приложение»

## Резюме для следующей фазы

### Выбранная идея: WordPulse (рабочее название WordMaster)

**Что это:** Мобильное приложение с набором ежедневных словесных мини-игр на русском и английском языках. Ключевая особенность — **познавательный контент**: после каждого задания пользователь узнаёт интересный факт (этимология, история слова, связь между языками, культурный контекст).

**Форматы мини-игр (MVP):**
1. **Анаграммы** — составь слово из перемешанных букв → факт об этимологии слова
2. **Составь слова** — из набора букв составь как можно больше слов → «Знаешь ли ты, что...»
3. **Цепочка слов** — измени одну букву, чтобы получить новое слово → связь между словами
4. **Поиск слов** — найди слова на сетке букв (word search) → тематические факты
5. **Угадай слово** — формат Wordle (5 попыток, подсказки цветом) → история слова

**Познавательный слой (ключевое отличие от конкурентов):**
- Каждый вопрос/задание сопровождается интересным **достоверным** фактом и (опционально) тематической иллюстрацией
- Каждый факт имеет **ссылку на источник** (Wikipedia, энциклопедия, научная книга) — факты НИКОГДА не выдумываются
- Ежедневные наборы имеют тему дня (напр. «Слова из древнегреческого», «Космическая лексика», «Еда мира»)
- Контент генерируется AI с обязательным указанием источника → владелец верифицирует факт и источник → одобряет
- Иллюстрации генерируются AI в едином стиле, владелец одобряет перед публикацией
- Пользователь чувствует, что не просто играет, а **узнаёт реальные факты** каждый день
- В приложении у каждого факта отображается источник (напр. «Источник: Wikipedia» или «Из книги: ...») — это повышает доверие

**Система антиповторов (ключевая UX-механика):**
- Сервер хранит историю: какие вопросы/факты пользователь уже видел и как ответил
- **Ответил правильно** → вопрос/факт больше НИКОГДА не показывается этому пользователю
- **Ответил неправильно** → вопрос может повториться, но не раньше чем через 7+ дней (элемент spaced repetition)
- **Не встречал** → вопрос доступен для показа
- Это гарантирует, что пользователь всегда видит **свежий контент**, а не одни и те же факты
- Требует достаточного пула вопросов: минимум 500+ одобренных вопросов к запуску

**Режимы:**
- Ежедневный набор (5 мини-игр, одинаковый для всех, один раз в день, с темой дня)
- Бесконечный режим (случайные головоломки без ограничений)

**Монетизация:** AdMob (баннеры + interstitial + rewarded video). Целевой eCPM blend: $1.5-3.0. Целевой DAU для $300/мес: 1,000-2,200.

**Целевая аудитория:** 18-45 лет, любители головоломок, изучающие языки, любознательные люди, RU + EN.

**Backend для MVP: ДА (NestJS + PostgreSQL + Prisma):**
- API для контента (вопросы, наборы, факты, категории, иллюстрации)
- AI-генерация контента (LLM для вопросов/фактов, image model для иллюстраций)
- Система антиповторов: правильно ответил → больше не видит этот вопрос
- Прогресс и статистика пользователей
- Глобальные лидерборды
- Push-уведомления
- S3-совместимое хранилище для картинок (Cloudflare R2)

**Web для MVP: ДА — Приватная админ-панель (React + Vite):**
- AI-генерация пачек вопросов → ручная модерация (одобрить/отклонить/редактировать)
- AI-генерация иллюстраций к вопросам → предпросмотр, перегенерация, загрузка своих
- Формирование и планирование ежедневных наборов

**Платформы:** iOS (App Store), Android (Google Play, RuStore, Huawei AppGallery).
