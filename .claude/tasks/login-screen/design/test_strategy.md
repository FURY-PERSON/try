# Стратегия тестирования: Экран логина (mobile)

## Критерии приёмки

1. Экран логина отображается при запуске приложения без авторизации
2. Поля "Логин" и "Пароль" принимают ввод
3. Кнопка "Войти" отправляет данные на API
4. При успешном логине — редирект на главный экран (tabs)
5. При ошибке — отображается сообщение об ошибке
6. Валидация: пустые поля не отправляются
7. При наличии сохранённой сессии — сразу показывается главный экран (без мелькания login)
8. Клавиатура не перекрывает форму (KeyboardAvoidingView)

## Ручное тестирование

### Happy path
- [ ] Запустить приложение без сохранённой сессии → видим экран логина
- [ ] Ввести валидный email и пароль → нажать "Войти" → попадаем на главный экран
- [ ] Закрыть и открыть приложение → сразу главный экран (сессия персистится)

### Error cases
- [ ] Ввести неверный пароль → сообщение об ошибке
- [ ] Оставить поля пустыми → нажать "Войти" → валидация не пропускает
- [ ] Отключить сеть → нажать "Войти" → корректная обработка ошибки

### UX
- [ ] Клавиатура не перекрывает поля ввода
- [ ] Пароль скрыт точками, toggle видимости работает
- [ ] Кнопка показывает loading state при отправке
- [ ] Safe area учитывается (notch, home indicator)

### Edge cases
- [ ] Перезапуск приложения во время запроса логина
- [ ] Быстрый двойной тап на "Войти" — не отправляет два запроса
