# Стратегия тестирования: Валюта "Щит"

## Критерии приёмки

1. Новый пользователь получает 5 щитов при регистрации
2. Баланс щитов отображается в header на home screen
3. По нажатию на баланс — модалка с описанием и кнопкой "Посмотреть видео"
4. В игре — иконка щита справа над карточками
5. При нажатии — щит активируется визуально
6. Неправильный ответ с активным щитом → streak сохранён + анимация
7. Бе�� щитов при нажатии — предложение посмотреть видео
8. Daily set: надпись про +3 щита за ≥50%
9. Guideline при первом сбросе стрика и восстановлении
10. +1 щит за каждые 10 в стрике
11. Серверная валидация: невозможно использовать щит с нулевым балансом

## Unit тесты (server)

### ShieldsService
- `useShield()` — успешное списан��е
- `useShield()` — ошибка при 0 shields (BadRequestException)
- `addShields()` — начисление корректной суммы
- `initializeShields()` — начисление 5 для нового пользователя

### QuestionsService.answerQuestion
- Неправильный ответ + useShield=true + shields>0 → streak сохранён, shields-=1
- Неправильный ответ + useShield=true + shields=0 → streak сброшен, shield not used
- Правильный ответ + useShield=true → shield не расходуется
- Streak кратный 10 → shields+=1

### DailySetsService.submitDailySet
- ≥50% правильных → +3 shields
- <50% правильных → +0 shields
- Shield usage в results корректно обрабатывается

## Integration тесты (server)
- Полный цикл: register → check shields=5 → answer wrong with shield → shields=4, streak intact
- Rewarded video endpoint: POST /shields/reward → shields += 2
