# Воркфлоу разработки фич с Claude Code

## Быстрый старт

**Один промпт — полный цикл:**
```
/feature task.md
/feature добавить систему достижений
```

Агент проведёт через все этапы (research → design → plan → implementation), останавливаясь на ревью после каждого. Тебе нужно только отвечать на вопросы и давать go/правки.

**Отдельные шаги** (если нужен контроль над каждым этапом):
```
/research <путь-к-файлу-задачи или описание>
/design <feature-name>
/plan <feature-name>
/implement <feature-name>
```

**Создать задачу из контекста разговора:**
```
/task добавить пагинацию в список вопросов
/task баг: daily set не генерируется
```

---

## Как это работает

### 1. Подготовь задачу

**Из файла** — создай файл с описанием:
```bash
echo "Добавить систему достижений для пользователей" > task.md
```

**Или просто опиши** в аргументе:
```
/feature добавить систему достижений для пользователей
```

### 2. Запусти `/feature`

```
/feature task.md
```

### 3. Что будет происходить

**Research** — агент задаст уточняющие вопросы, исследует кодовую базу, покажет сводку.
→ Ты проверяешь и говоришь "продолжить" или "доисследуй X"

**Design** — создаст архитектуру, API контракты, решения, стратегию тестирования.
→ Ты проверяешь файлы в `.claude/tasks/<feature>/design/` и говоришь "продолжить" или даёшь правки

**Plan** — разобьёт на фазы (≤ 5-7 файлов), построит граф зависимостей.
→ Ты проверяешь план и говоришь "продолжить" или корректируешь

**Implementation** — для каждой фазы: код → code quality ревью → compliance ревью → билд/тесты → коммит.
→ По завершении проверяешь результат

### 4. Финализация

```
/simplify          # пост-имплементационный ревью (опционально)
git push           # пушишь самостоятельно
```

## Структура артефактов

```
.claude/tasks/<feature-name>/
  research/
    research_result.md
  design/
    architecture.md
    contracts.md
    decisions.md
    test_strategy.md
  plan/
    plan.md
  status.md
```

---

## Когда можно пропустить шаги

| Размер задачи | Что запускать |
|---------------|---------------|
| Мелкий фикс (1-2 файла) | Работай напрямую, без пайплайна |
| Небольшая задача (3-5 файлов) | `/research` → сразу реализация |
| Средняя фича | `/research` → `/design` → `/plan` → `/implement` |
| Крупная фича | Полный цикл + `/simplify` |

---

## Возврат к предыдущему шагу

Если на ревью нашёл проблемы:
```
# вернуться к дизайну и внести правки
/design <feature-name>
> Измени architecture.md: <описание правок>

# после правок — перезапустить план и реализацию
/plan <feature-name>
/implement <feature-name>
```

Артефакты обновляются, а не пересоздаются с нуля.
