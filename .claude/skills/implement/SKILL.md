# Implement: Реализация фичи

## Входные данные

Feature: $ARGUMENTS

1. Прочитай `.claude/tasks/$0/plan/plan.md`
2. Прочитай все файлы в `.claude/tasks/$0/design/`
3. Прочитай `.claude/tasks/$0/research/research_result.md`
4. Прочитай `.claude/tasks/$0/status.md` — убедись что план завершён
5. Прочитай `.claude/rules/git-conventions.md`
6. Определи область из `status.md` и прочитай соответствующие правила:
   - `server-conventions.md` для server
   - `web-conventions.md` для web
   - `mobile-conventions.md` для mobile

Если файлы не найдены — сообщи пользователю и предложи пройти предыдущие шаги.

## Роль: Тимлид

Ты выступаешь как тимлид. Твои обязанности:
1. Прочитать и понять задачу, дизайн, план
2. Выполнять фазы в правильном порядке (по зависимостям)
3. Для каждой фазы организовать цикл: разработка -> ревью -> тестирование
4. Коммитить после успешного прохождения всех проверок
5. Эскалировать пользователю при превышении лимита итераций

## Пайплайн выполнения фазы

Для каждой фазы из плана выполни последовательно:

### 1. Разработка

Реализуй фазу согласно плану:
- Следуй описанию из "Что сделать"
- Используй паттерны из research_result.md

### 2. Code Quality ревью

Запусти агент `code-quality-reviewer` через Agent tool для проверки:
- Качество, безопасность, зависимости, типы

### 3. Compliance ревью

Запусти агент `compliance-reviewer` через Agent tool для проверки:
- Соответствие плану, дизайну, архитектуре

### 4. TypeScript проверка

```bash
# Для server:
cd server && npx tsc --noEmit

# Для web:
cd web && npx tsc --noEmit

# Для mobile:
cd mobile && npx tsc --noEmit
```
Если есть TS ошибки — исправить ДО коммита.

### 5. Тестирование

```bash
# Для server:
cd server && npm test

# Для web:
cd web && npm run build

# Для mobile:
cd mobile && npm run lint
```

### 6. MCP-тестирование (ОБЯЗАТЕЛЬНО для UI-фаз)

В зависимости от области:
- **mobile** — использовать mobile-mcp для визуальной проверки экрана
- **web** — использовать playwright-mcp для проверки в браузере

Если нужный MCP не подключён — спросить пользователя: пропустить или подключить. НЕ пропускать молча.

### 7. Коммит фазы

**ОБЯЗАТЕЛЬНО** создать коммит после каждой успешной фазы:
- Формат: `<type>(<scope>): <описание на русском>` (из `.claude/rules/git-conventions.md`)
- **Без Co-Authored-By** — НЕ добавлять `Co-Authored-By: Claude`
- НЕ делать push

### 8. Обработка проблем

Если найдены проблемы на любом шаге:
- Исправь проблемы
- Повтори цикл с шага 2
- **Максимум 3 итерации на фазу**
- После 3-й неудачи — ОСТАНОВИСЬ и сообщи пользователю

## Финальная проверка

После всех фаз:
```bash
cd server && npm run build && npm test
cd web && npm run build
cd mobile && npm run lint
```

## Завершение

1. Обнови `.claude/tasks/$0/status.md` со статусом "реализация завершена"
2. Выведи сводку:
   - Количество выполненных фаз
   - Количество коммитов
   - Список изменённых файлов
   - Результаты финального билда и тестов
3. Предложи пользователю запустить `/simplify` для пост-имплементационного ревью
4. Напомни что push нужно сделать самостоятельно
