# Research: Исследование кодовой базы

## Входные данные

Задача: $ARGUMENTS

## Шаг 0: Предусловия

1. Определи `feature-name` из задачи (короткое имя на английском, kebab-case)
2. Создай директорию `.claude/tasks/{feature-name}/research/`
3. Если задача — путь к файлу, прочитай файл

## Шаг 0.5: Определение области

Определи какие части проекта затрагивает задача. Если неясно — ОБЯЗАТЕЛЬНО спроси:

> Какие части проекта затрагивает эта задача?
> 1. **server** — NestJS API (server/)
> 2. **web** — Admin panel (web/)
> 3. **mobile** — Expo приложение (mobile/)
> 4. **full-stack** — несколько частей

## Шаг 1: Валидация входных данных

Проведи интервью с пользователем по каждому аспекту задачи до достижения общего понимания.

Используй AskUserQuestion для уточнения неясных требований. Если вопрос можно прояснить исследованием кодовой базы — исследуй код вместо того чтобы спрашивать.

## Шаг 2: Исследование

Проанализируй задачу и определи области кодовой базы для исследования:
- Какие модули/сервисы затрагиваются?
- Какие существующие паттерны используются?
- Какие зависимости и интеграции задействованы?
- Какие Prisma-модели связаны с задачей?

Для каждой области запусти саб-агент `researcher` параллельно через Agent tool:
```
Agent(subagent_type="Explore", model="haiku", prompt="Исследуй: {конкретная подзадача}")
```

## Шаг 3: Консолидация

Объедини результаты всех исследований в `.claude/tasks/{feature-name}/research/research_result.md`.

## Формат результата

```markdown
# Исследование: {название задачи}

## Задача
{краткое описание задачи}

## Область: {server | web | mobile | full-stack}

## Текущее состояние кодовой базы

### Затрагиваемые модули
- `server/src/modules/{name}`: {что делает, как связан с задачей}
- `web/src/pages/{name}`: {что делает}
- `mobile/src/features/{name}`: {что делает}

### Существующие паттерны
- {описание паттерна}: {пример из кода с путём к файлу}

### Зависимости и интеграции
- {зависимость}: {как используется}

### Ключевые файлы
- `{путь}:{строка}` — {описание}

### Модель данных (Prisma)
- {существующие модели, связанные с задачей}

### API контракты
- {существующие эндпоинты, связанные с задачей}
```

## КРИТИЧЕСКИ ВАЖНО

Результат содержит ТОЛЬКО факты о текущем состоянии кодовой базы.

**ЗАПРЕЩЕНО включать:**
- Рекомендации по улучшению
- Предложения по рефакторингу
- Оценки качества существующего кода

## Завершение

1. Создай/обнови `.claude/tasks/{feature-name}/status.md` со статусом "исследование завершено"
2. Сообщи пользователю что исследование готово и предложи проревьюить `research_result.md`
3. НЕ переходи к следующему шагу (дизайн) без явного подтверждения пользователя
