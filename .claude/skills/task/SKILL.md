# /task

Создаёт задачу или баг из контекста текущего разговора, сохраняя в `.claude/tasks/`.

Всё общение с пользователем — на русском языке.

## Шаблоны

- **Задача** (Feature / Improvement / Chore) → `.claude/templates/task-template.md`
- **Баг** (Bug) → `.claude/templates/bug-template.md`

Шаблон выбирается автоматически по типу. Ключевые слова «баг», «сломалось», «не работает», «ошибка» → шаблон бага.

## Аргументы

- Короткое описание задачи (опционально — берётся из контекста разговора). Примеры:
  - `/task добавить пагинацию в список вопросов`
  - `/task баг: daily set не генерируется`

## Процесс

### 1. Собрать контекст

Проанализировать текущий разговор и определить:
- **Что нужно сделать** — суть задачи из обсуждения или аргумента
- **Ценность** — зачем это нужно (если обсуждалось)
- **Область** — server / web / mobile (из контекста разговора или спросить)
- **Технические детали** — файлы, модули (из исследования в разговоре)
- **Тип** — задача (feature/improvement/chore) или баг

### 2. Уточнить детали

Использовать `AskUserQuestion` для заполнения недостающей информации.
Минимизировать вопросы, если контекст достаточен — предложить значения по умолчанию.

### 3. Составить описание

Выбрать шаблон по типу и заполнить из контекста разговора.

### 4. Показать превью

Показать пользователю составленную задачу и спросить:
- Сохранить как есть
- Отредактировать (пользователь вносит правки)

### 5. Сохранить

Сохранить задачу в `.claude/tasks/{task-name}.md` (kebab-case).

## Правила

- Описание задачи всегда на русском
- Не выдумывать детали — использовать только то, что было в разговоре
- Область обязательна — определить из контекста или спросить
