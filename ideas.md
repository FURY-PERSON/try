# Fact Front: Механики ретеншена и виральности

## Текущее состояние (TL;DR)

Сильная база: стрики с 7-уровневой визуализацией, лидерборд, heatmap, шеринг emoji-сеткой. Ключевые пробелы: **нет социального графа, нет progression system, нет FOMO-триггеров, шеринг пассивный**.

---

## QUICK WINS (низкая сложность, высокий эффект)

### 1. «Факт дня» с Wordle-стилем шеринга

**Механика:** Выделить 1 вопрос из Daily Set как «Факт дня» — самый удивительный/контринтуитивный. После ответа показать экран: «87% людей ответили неправильно! А ты?» + кнопка шеринга с текстом типа:

```
Факт дня 🧠
"Медведи впадают в спячку" — правда или миф?
Я угадал ✅ (только 13% знают!)
factfront.app
```

**Почему работает:** Wordle стал вирусным не из-за игры, а из-за формата шеринга — минимальный, интригующий, без спойлеров. Факт создаёт conversation starter: «А ты знал, что...?»

**Сложность:** Низкая — добавить поле `isFactOfDay` к DailySetQuestion, отдельный экран после результатов.

**Эффект:** Виральность +++ | Ретеншен +

**Приоритет:** MUST-HAVE

### 2. Streak Freeze (страховка стрика)

**Механика:** Дать 1 бесплатный «замороженный день» в неделю. Если пропустил — стрик не обнуляется, а ставится на паузу. Дополнительные freeze можно получить за просмотр rewarded video или за 7-дневный стрик.

**Почему работает:** Duolingo доказал, что streak freeze — ключевая механика удержания. Без неё стрик вызывает тревогу → юзер бросает вообще после первого пропуска (эффект «what-the-hell»). С freeze — юзер знает, что 1 пропуск не фатален, и возвращается.

**Сложность:** Низкая — поле `streakFreezeCount` + `lastFreezeUsed` на User, логика в daily set submission.

**Эффект:** Ретеншен D7 +++ | D30 +++

**Приоритет:** MUST-HAVE

---

### 3. Push-уведомление «Стрик в опасности 🔥»

**Механика:** Вечером (за 2-3 часа до полуночи) отправлять push: «Твой стрик 23 дня сгорит через 3 часа! 🔥» — только если юзер сегодня НЕ играл. Персонализировать: чем длиннее стрик, тем эмоциональнее текст.

**Почему работает:** Loss aversion — потеря стрика в 30 дней ощущается в 2-3x сильнее, чем радость от его набора (Kahneman). Duolingo's «sad owl» — один из самых эффективных push в мобайле.

**Сложность:** Низкая — серверный cron (`@nestjs/schedule` уже есть), проверка `UserQuestionHistory` за сегодня, отправка push по `pushToken`.

**Эффект:** Ретеншен D1 +++ | D7 ++

**Приоритет:** MUST-HAVE

---

### 4. Post-game «А ты знал?» карточка

**Механика:** После каждого неправильного ответа (или в конце игры) показывать красивую карточку с объяснением факта + источником. Кнопка «Поделиться фактом» генерирует карточку-картинку (а не текст) для Instagram Stories / Telegram.

**Почему работает:** Контент — это валюта. Люди шерят то, что делает их умнее в глазах других. «Посмотри, я сегодня узнал, что...» — естественнее, чем «Я набрал 15/20».

**Сложность:** Средняя — генерация изображения (react-native-view-shot), шеринг картинки через Share API.

**Эффект:** Виральность +++ | Ретеншен +

**Приоритет:** MUST-HAVE

### 5. Лиги Duolingo-стиля

**Механика:** Разделить всех активных игроков на лиги по 30-50 человек (Bronze → Silver → Gold → Platinum → Diamond → Obsidian). Каждую неделю: топ-5 повышаются, нижние 5 понижаются. Отдельный таб в лидерборде.

**Почему работает:** Глобальный лидерборд демотивирует — ты никогда не будешь в топ-100. Лига из 30 человек создаёт: (1) ощущение «я могу выиграть», (2) социальное сравнение с peer group, (3) страх вылета. У Duolingo лиги увеличили DAU на 17%.

**Сложность:** Средняя — еженедельный cron для перераспределения, новая таблица `League` + `LeagueEntry`, UI таба.

**Эффект:** Ретеншен D7 +++ | D30 +++

**Приоритет:** MUST-HAVE

## MEDIUM EFFORT, HIGH IMPACT

### 6. Challenge a Friend (асинхронный PvP)

**Механика:** Кнопка «Вызвать друга» генерирует ссылку. Друг проходит те же 10 вопросов. Оба видят результат: кто ответил правильнее и быстрее. Не требует аккаунта — работает через deep link.

**Почему работает:** Kahoot доказал, что quiz + соревнование = магия. Асинхронность решает проблему координации. Deep link = zero-friction онбординг нового юзера. Это главный viral loop: играл → вызвал друга → друг установил → вызвал своего друга.

**Сложность:** Средняя — новая сущность `Challenge` (набор вопросов + результаты двух игроков), deep linking (уже есть `factfront.app`), UI экрана вызова.

**Эффект:** Виральность +++++ | Ретеншен ++

**Приоритет:** MUST-HAVE

### 7. Система достижений (Achievements)

**Механика:** 30-50 достижений с иконками:
- **Стрик-серия:** 7 дней, 30 дней, 100 дней, 365 дней
- **Точность:** 100% в Daily Set, 10 подряд правильных, 50 подряд
- **Объём:** 100/500/1000/5000 фактов
- **Категории:** «Мастер науки» (все вопросы в категории), «Эрудит» (все категории)
- **Скорость:** Ответ менее 2 секунд, весь Daily Set за 60 секунд
- **Социальные:** Первый шер, 10 шеров, выиграть challenge

Показывать прогресс-бары. Unlock-анимация с конфетти.

**Почему работает:** Variable reward (Skinner box) — юзер не знает, какой ачивмент «выпадет» следующим. Ачивменты дают micro-goals между macro-goals (стрик/лидерборд). Коллекционирование — мощный драйвер (Pokemon effect).

**Сложность:** Средняя — новая таблица `Achievement` + `UserAchievement`, триггерная логика в answer/submit flow, UI витрины.

**Эффект:** Ретеншен D30 +++ | Виральность + (шеринг ачивок)

**Приоритет:** MUST-HAVE

### 8. «Вернись и исправь» — Revenge Mode

**Механика:** На следующий день после Daily Set показать блок: «Вчера ты ошибся в 5 вопросах. Хочешь разобраться?» → мини-сессия из вчерашних ошибок с подробными объяснениями. За прохождение — бонусные очки.

**Почему работает:** (1) Создаёт дополнительную точку входа в приложение. (2) Spaced repetition — доказанно лучший способ запоминания. (3) Юзер чувствует прогресс: «вчера не знал, сегодня знаю».

**Сложность:** Низкая — данные уже есть в `UserQuestionHistory`, нужен новый блок на Home + мини-игровой экран.

**Эффект:** Ретеншен D1 ++ | D7 ++

**Приоритет:** NICE-TO-HAVE (но очень близко к must-have)

### 9. Тематические ивенты с таймером

**Механика:** Еженедельные/ежемесячные тематические события: «Неделя космоса 🚀» (7 дней, специальные вопросы про космос, отдельный мини-лидерборд, уникальный badge). FOMO-таймер: «Осталось 2 дня 14 часов».

**Почему работает:** BeReal/Fortnite — ограниченность по времени создаёт urgency. Тематичность даёт повод для контента в соцсетях. Уникальный badge = коллекционный элемент, который больше нельзя получить.

**Сложность:** Средняя — расширить модель `Collection` таймером и бейджем, админ-панель для создания ивентов, UI на Home.

**Эффект:** Ретеншен D7 ++ | Виральность ++

**Приоритет:** NICE-TO-HAVE

### 10. Onboarding → Hook цепочка

**Механика:** Первые 3 дня нового юзера — специальный flow:
- **День 0:** Короткий квиз из 5 вопросов (не 20!) → вау-результат → «Приходи завтра за новыми»
- **День 1:** Push «Твой первый стрик! Не потеряй 🔥» → Daily Set
- **День 2:** Push «Стрик 2 дня! На 3-й день откроется сюрприз» → ачивмент «Три дня подряд»
- **День 3:** Разблокировка Challenge a Friend

**Почему работает:** Nir Eyal's Hook Model — первые 3 сессии критичны для формирования привычки. Уменьшение friction в день 0 (5 вопросов вместо 20) повышает completion rate. Escalating rewards создают momentum.

**Сложность:** Средняя — отдельный onboarding flow, серверная логика для «дня юзера», условные push-уведомления.

**Эффект:** Ретеншен D1 ++++ | D7 +++

**Приоритет:** MUST-HAVE

## HIGHER EFFORT, STRATEGIC

### 11. Live Daily Tournament (Micro-HQ Trivia)

**Механика:** Раз в день в фиксированное время (например, 20:00 MSK) — 5-минутный турнир. Все отвечают на одни вопросы одновременно. Реалтайм-лидерборд. Топ-10 получают специальный badge дня.

**Почему работает:** HQ Trivia на пике собирало 2M+ одновременно. Фиксированное время = ритуал. Синхронность создаёт ощущение community. «Сегодня я был 3-й из 500!» — мощный повод для шера.

**Сложность:** Высокая — WebSocket/SSE для реалтайма, синхронизация, серверная нагрузка.

**Эффект:** Ретеншен D1 ++++ | Виральность ++++

**Приоритет:** ЭКСПЕРИМЕНТ (после достижения 10K DAU)

### 12. «Ты умнее, чем думаешь» — IQ/Knowledge Score

**Сложность:** Средняя — агрегация по категориям уже частично есть, нужна формула индекса + UI radar chart.

## «ИЗЮМИНКА» — уникальная фича

### 13. «Debate Mode» — Факт-дебаты / «Спорим, ошибёшься»

**Механика:** Особый тип вопроса: показывается спорное утверждение. Юзер голосует «Правда/Миф» и видит, как проголосовали ВСЕ игроки (live poll). Затем открывается правильный ответ + статистика: «67% ошиблись!».

Ключевое: после голосования юзер может нажать «Спорим, ты тоже ошибёшься?» → шерится ссылка с вопросом (без ответа) другу.

**Почему это «изюминка»:**
- У Wordle есть ежедневный пазл, у Duolingo — сова, у Fact Front будет **«Ты тоже ошибёшься»** — вирусный challenge-формат
- Спорные факты = обсуждение = органический word-of-mouth
- Live poll создаёт ощущение community без сложной социальной инфры
- Формат «prediction → reveal» = дофаминовый цикл (как скретч-лотерея)

**Сложность:** Средняя — новый тип вопроса с агрегацией голосов, real-time не нужен (обновлять poll каждые 30 сек), deep link для шеринга.

**Эффект:** Виральность +++++ | Ретеншен +++

**Приоритет:** MUST-HAVE (это ваш дифференциатор)

### 14. Emoji-реакции на факты + социальная статистика

**Механика:** После ответа на вопрос (на экране объяснения) показать ряд emoji-реакций: 🤯 😂 😱 🤔 💡 🥱. Юзер тапает одну — его голос сохраняется. На этом же экране (или при повторном просмотре факта) показывается агрегат: «72% считают это смешным 😂», «45% были в шоке 🤯».

**Детали реализации:**
- После свайпа карточки и показа объяснения — внизу карточки ряд из 5-6 emoji-кнопок
- Один тап = голос записан, кнопка подсвечивается, появляется мини-анимация
- Под кнопками — top-1 или top-2 реакции с процентами: «🤯 54% · 😂 23%»
- Данные обновляются асинхронно (fire-and-forget POST, не блокирует геймплей)
- При шеринге факта добавлять: «🤯 54% игроков были в шоке!»

**Серверная часть:**
- Новая таблица `QuestionReaction`: `userId`, `questionId`, `emoji`, `createdAt` (unique: userId+questionId)
- Эндпоинт `POST /questions/:id/reaction` — сохранить/обновить реакцию
- Агрегат считать через `GROUP BY emoji` + `COUNT` (кешировать в Redis или в поле `reactionStats` на Question)
- Возвращать агрегат в ответе на `answerQuestion` или отдельным `GET /questions/:id/reactions`

**Почему работает:**
- **Micro-interaction = engagement.** Тап на emoji — zero-effort действие, но создаёт ощущение участия и community
- **Social proof.** «72% были в шоке» — делает факт ещё интереснее, хочется поделиться
- **Контент для шеринга.** «Факт, от которого 80% людей в шоке» — кликбейт, который работает органически
- **Data goldmine.** Реакции показывают, какие факты вызывают эмоции → приоритизировать их для Daily Set и Fact of the Day
- **Прецедент:** Netflix thumbs up/down, YouTube like, Spotify heart — одно действие, но создаёт привязку

**Сложность:** Низкая — одна таблица, один эндпоинт, ряд кнопок на экране feedback'а

**Эффект:** Ретеншен ++ | Виральность ++ (усиливает шеринг фактов)

**Приоритет:** NICE-TO-HAVE (но отлично сочетается с #1 Факт дня и #4 Карточка-картинка)

## Переработка стриков: Daily Streak

### Проблема

Сейчас стрик = количество правильных ответов подряд (`currentAnswerStreak`). Один неправильный ответ — стрик сбрасывается. Это:
- Не мотивирует заходить каждый день (стрик не привязан к дням)
- Быстро ломается и демотивирует
- Не создаёт привычку ежедневного возврата

### Решение: два типа стриков

**1. Daily Streak (основной, Duolingo-стиль)**
- Считает дни подряд, в которые юзер сыграл хотя бы 1 игру
- Сбрасывается только если пропустил целый день
- Показывается на Home, в профиле, в шеринге, в лидерборде
- Milestone-празднования на 7, 30, 100, 365 дней

**2. Answer Streak (вторичный, оставить как есть)**
- Правильных ответов подряд — показывать во время игры и на результатах
- Визуальный фидбек в реальном времени (уже реализован через StreakBadge тиры)

### Что менять

#### Prisma Schema (`server/prisma/schema.prisma`)
Добавить поля в User:
```
dailyStreak        Int       @default(0)    // Дней подряд с активностью
bestDailyStreak    Int       @default(0)    // Рекорд daily streak
streakFreezeCount  Int       @default(1)    // Кол-во freeze (1 бесплатный)
lastStreakFreezeUsed DateTime?              // Когда использован последний freeze
```
Существующие поля `currentStreak`, `bestStreak` — переименовать НЕ нужно, просто перестать использовать как основной стрик. `lastPlayedDate` уже есть — использовать для расчёта daily streak.

#### Server: логика daily streak

В `daily-sets.service.ts` и `collections.service.ts` при submit:

```typescript
function calculateDailyStreak(user: User): { dailyStreak: number; bestDailyStreak: number; freezeUsed: boolean } {
  const today = new Date();
  today.setUTCHours(0, 0, 0, 0);

  const lastPlayed = user.lastPlayedDate;
  if (!lastPlayed) {
    // Первая игра
    return { dailyStreak: 1, bestDailyStreak: 1, freezeUsed: false };
  }

  const lastPlayedDay = new Date(lastPlayed);
  lastPlayedDay.setUTCHours(0, 0, 0, 0);

  const diffDays = Math.floor((today.getTime() - lastPlayedDay.getTime()) / (1000 * 60 * 60 * 24));

  if (diffDays === 0) {
    // Уже играл сегодня — стрик не меняется
    return { dailyStreak: user.dailyStreak, bestDailyStreak: user.bestDailyStreak, freezeUsed: false };
  }

  if (diffDays === 1) {
    // Играл вчера — инкремент
    const newStreak = user.dailyStreak + 1;
    return { dailyStreak: newStreak, bestDailyStreak: Math.max(newStreak, user.bestDailyStreak), freezeUsed: false };
  }

  if (diffDays === 2 && user.streakFreezeCount > 0) {
    // Пропустил 1 день, есть freeze — использовать
    const newStreak = user.dailyStreak + 1;
    return { dailyStreak: newStreak, bestDailyStreak: Math.max(newStreak, user.bestDailyStreak), freezeUsed: true };
  }

  // Пропустил >1 дня (или нет freeze) — сброс
  return { dailyStreak: 1, bestDailyStreak: user.bestDailyStreak, freezeUsed: false };
}
```
